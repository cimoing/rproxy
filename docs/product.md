# RProxy 产品文档

## 1. 产品概述

RProxy 是一款跨平台 GUI 代理工具，面向需要统一管理网络代理、自动路由和多协议连接的桌面用户。产品底层核心使用 Rust 开发，提供高性能、低资源占用和跨平台能力；图形界面使用 Slint 实现，提供轻量、原生、响应快速的桌面体验。

RProxy 的核心目标是让用户用最少配置完成代理接入、路由分流、PAC 生成和多配置切换，同时保留高级用户需要的协议、路由和 Tun 模式控制能力。

## 2. 产品定位

### 2.1 产品形态

- 跨平台桌面 GUI 应用。
- 本地代理客户端与路由控制工具。
- 支持系统代理、Tun 模式代理、PAC 自动生成和多配置管理。

### 2.2 目标用户

- 需要稳定代理连接和规则分流的个人用户。
- 需要在多个代理配置之间切换的开发者、运维人员和技术用户。
- 需要更低延迟、更高性能和更强可控性的高级用户。

### 2.3 核心价值

- 一个应用完成代理连接、路由分流、系统代理和 PAC 管理。
- 支持多协议接入，降低用户迁移和配置成本。
- 自动路由与自动 PAC 减少手工维护规则的复杂度。
- Rust 核心保障代理转发、规则匹配和 Tun 处理的性能与稳定性。
- Slint UI 提供一致、轻量的跨平台桌面体验。

## 3. 功能范围

### 3.1 跨平台 GUI 代理

RProxy 提供桌面图形界面，支持用户完成以下操作：

- 查看当前代理运行状态。
- 启动、停止和重启代理服务。
- 查看当前活动配置。
- 查看当前代理模式、监听端口和路由策略。
- 切换代理配置。
- 管理代理节点和连接信息。
- 查看运行日志和错误提示。

首批支持平台：

- Windows。

后续可扩展平台：

- macOS。
- Linux。

### 3.2 Tun 模式代理

Tun 模式用于接管系统或指定流量，实现更完整的透明代理能力。

核心能力：

- 创建和管理虚拟网络接口。
- 将流量导入代理核心。
- 按路由规则决定直连、代理或阻断。
- 支持 IPv4，后续可扩展 IPv6。
- 支持与自动路由模块联动。

用户场景：

- 应用不支持系统代理时，仍可通过 Tun 模式接入代理。
- 需要按域名、IP、地理位置或规则集进行统一分流。
- 需要对系统级流量进行更完整的代理控制。

### 3.3 多协议支持

RProxy 支持多种代理协议，并预留协议扩展能力。

首批协议：

- HTTP。
- SOCKS。
- VLESS，首批支持 TCP、TLS 与 WebSocket 传输。
- VMess，首批支持 legacy TCP/TLS/WebSocket，`security: none`。

后续可扩展：

- Shadowsocks。
- Trojan。
- WireGuard。
- Hysteria。

协议能力要求：

- 统一的协议配置模型。
- 统一的连接生命周期管理。
- 统一的健康检查与错误展示。
- 支持按节点、按配置或按路由策略选择出口。

### 3.4 自动路由

自动路由用于根据规则决定流量处理方式。

路由动作：

- Proxy：走代理。
- Direct：直连。
- Block：阻断。

路由匹配维度：

- 域名。
- 域名后缀。
- IP 段。
- 端口。
- 协议类型。
- 规则集。
- 地理位置数据库。

路由策略：

- 全局代理。
- 全局直连。
- 自动分流。
- 自定义规则优先。

规则优先级：

1. 用户自定义规则。
2. 当前配置内置规则。
3. 规则集规则。
4. 默认策略。

### 3.5 自动 PAC

PAC 功能根据当前路由策略自动生成代理自动配置脚本。

核心能力：

- 根据路由规则生成 PAC。
- 本地启动 PAC HTTP 服务。
- 自动更新系统 PAC 地址。
- 配置切换后自动刷新 PAC。
- 路由规则变更后自动重新生成 PAC。

PAC 输出目标：

- 需要走代理的域名返回代理地址。
- 需要直连的域名返回 DIRECT。
- 阻断规则可返回不可达代理或按产品策略处理。

### 3.6 多配置管理

RProxy 支持用户维护多个代理配置，并在不同网络环境中快速切换。

配置能力：

- 创建、编辑、删除配置。
- 导入和导出配置。
- 设置默认配置。
- 一键切换当前活动配置。
- 配置节点、协议、路由、PAC 和 Tun 参数。
- 支持配置启用状态与备注信息。

配置来源：

- 本地手动创建。
- YAML 文件导入。
- 剪贴板导入。

后续可增加订阅格式兼容，例如 Clash、Sing-box 或自定义订阅格式。

配置数据建议包含：

- 配置名称。
- 节点列表。
- 代理协议参数。
- 路由规则。
- Tun 设置。
- PAC 设置。
- 系统代理设置。

## 4. 用户流程

### 4.1 首次启动

1. 用户打开 RProxy。
2. 应用检查系统权限和网络环境。
3. 用户导入或创建第一个配置。
4. 用户选择代理模式。
5. 用户启动代理。
6. 应用显示运行状态和当前路由策略。

### 4.2 日常使用

1. 用户打开应用。
2. 应用加载默认配置。
3. 用户点击启动。
4. 应用启动代理核心、系统代理、PAC 或 Tun。
5. 用户可查看连接状态、日志和流量信息。
6. 用户可随时切换配置或停止代理。

### 4.3 配置切换

1. 用户选择目标配置。
2. 应用停止当前配置相关运行资源。
3. 应用加载新配置。
4. 应用刷新路由、PAC 和代理监听。
5. 应用恢复代理运行状态。

## 5. 技术方案

### 5.1 技术栈

- 核心语言：Rust。
- GUI 框架：Slint。
- 网络代理核心：Rust async runtime。
- 配置格式：YAML。
- 本地持久化：文件存储，后续可扩展 SQLite。
- 首批支持平台：Windows。
- 地理规则数据：使用 v2fly/domain-list-community 的 `dlc.dat`，首版暂不支持在线更新；文件缺失时回退到内置种子数据。

### 5.3 核心模块

#### GUI 模块

- 使用 Slint 构建主窗口、配置管理、日志面板和状态面板。
- 通过应用服务层调用核心能力。
- 不直接处理代理转发和复杂网络逻辑。

#### 配置模块

- 负责配置解析、校验、迁移和持久化。
- 提供统一配置模型。
- 支持多配置与当前活动配置。

#### 代理核心模块

- 负责本地监听、连接转发和协议实现。
- 提供 HTTP、SOCKS、VLESS 等协议支持。
- 对上层暴露统一启动、停止、重载和状态查询接口。

#### 路由模块

- 负责路由规则加载与匹配。
- 为代理核心、Tun 模块和 PAC 模块提供统一决策。
- 输出 Direct、Proxy 或 Block。

#### PAC 模块

- 根据路由规则生成 PAC 脚本。
- 提供本地 PAC 服务。
- 与系统代理模块联动。

#### Tun 模块

- 创建虚拟网卡并接管流量。
- 将流量交给路由模块决策。
- 将代理流量转发到代理核心。

#### 平台适配模块

- Windows、macOS、Linux 的系统代理设置。
- Tun 权限、驱动或虚拟接口差异处理。
- 系统托盘、开机启动和通知能力。

## 6. 配置模型草案

```yaml
profile:
  id: default
  name: Default
  enabled: true

nodes:
  - id: node-1
    name: Example VLESS
    protocol: vless
    server: example.com
    port: 443
    options:
      uuid: 00000000-0000-0000-0000-000000000000
      tls: true
      transport: websocket
      websocket:
        path: /proxy
        host: example.com

proxy:
  http_listen: 127.0.0.1:7890
  socks_listen: 127.0.0.1:7891

system:
  tray: true
  auto_start: false

tun:
  enabled: false
  interface_name: rproxy-tun
  auto_route: true

pac:
  enabled: true
  listen: 127.0.0.1:7892

routing:
  mode: auto
  default_action: proxy
  geosite:
    enabled: true
    auto_update: false
  rules:
    - type: domain_suffix
      value: example.cn
      action: direct
    - type: ip_cidr
      value: 10.0.0.0/8
      action: direct
```

## 7. 非功能需求

### 7.1 性能

- 代理转发低延迟。
- 路由匹配可支持大量规则。
- PAC 生成与配置切换不应阻塞 UI。
- 日志写入不应影响代理转发。

### 7.2 稳定性

- 代理核心异常退出后可提示并恢复状态。
- 配置加载失败时保留上一个可用配置。
- Tun 启动失败时清晰提示权限或环境问题。
- 配置切换时避免端口、接口和系统代理状态残留。

### 7.3 安全性

- 敏感字段本地存储时避免明文展示。
- 日志默认隐藏密钥、UUID、密码和令牌。
- PAC 服务默认只监听本地地址。
- 管理接口默认不暴露到公网。

### 7.4 可维护性

- GUI 与代理核心解耦。
- 协议通过适配层扩展。
- 路由规则与 PAC 生成共享同一决策模型。
- 平台相关能力集中在平台适配层。

## 9. 已确认产品决策

- 首批正式支持平台为 Windows。
- 配置文件使用 YAML 格式。
- VLESS 首批支持 TLS 与 WebSocket。
- Tun 模式作为第二阶段能力实现。
- 需要系统托盘与开机自启动配置。
- 需要内置 geosite 数据库，首版暂不支持在线更新。
- 后续可增加订阅格式兼容。

## 10. 待确认问题

- Windows 安装包格式选择，例如 MSI、NSIS 或 MSIX。
- geosite 数据库来源、许可证和构建期打包方式。
- VLESS WebSocket 是否需要支持自定义 Header。
- 系统代理是否需要支持自动恢复用户原始代理配置。
- 首版是否需要管理员权限引导流程。
